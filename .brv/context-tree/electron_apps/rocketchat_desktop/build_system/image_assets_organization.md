## Raw Concept
**Task:**
Static image assets organization for Rocket.Chat Electron app - structured directory layout for tray icons, app icons, and UI assets

**Files:**
- src/public/images/
- src/public/images/tray/
- src/public/images/tray/darwin/
- src/public/images/tray/linux/
- src/public/images/tray/win32/
- src/public/images/touch-bar/

**Flow:**
src/public/images/ (root) -> tray/ (platform-specific) -> {darwin, linux, win32}/ + touch-bar/ + icon.ico + file-icon.svg

**Timestamp:** 2026-04-04

## Narrative
### Structure
Image assets are organized in src/public/images/ with three main categories: (1) tray/ contains platform-specific system tray icons organized by OS (darwin/, linux/, win32/), (2) touch-bar/ contains macOS Touch Bar formatting button icons, (3) root level contains application icons and utility assets. All assets are generated by src/buildAssets.ts except for file-icon.svg which is static.

### Highlights
macOS tray icons: defaultTemplate.png/defaultTemplate@2x.png (24/48px), notificationTemplate.png/notificationTemplate@2x.png (24/48px). Windows tray icons: default.ico and notification badges (notification-1 through notification-9, notification-dot, notification-plus-9) at 16-256px multi-resolution ICO format. Linux tray icons: default.png/default@2x.png (64/128px) with notification-1 through notification-9, notification-dot, notification-plus-9 variants plus @2x versions. Touch Bar icons: bold.png, italic.png, strike.png, inline_code.png, multi_line.png for text formatting. Root assets: icon.ico (Windows app icon, 16-256px multi-res), file-icon.svg (download file type display).

### Rules
Rule 1: macOS tray icons use "Template" naming convention (defaultTemplate, notificationTemplate) for proper system rendering
Rule 2: Retina display support via @2x suffix on macOS and Linux assets
Rule 3: Windows tray uses single multi-resolution ICO format instead of separate files
Rule 4: All platform-specific tray icons include notification badge variants (1-10 and dot/plus-9)
Rule 5: Touch Bar icons are specific to macOS and used for text formatting operations
Rule 6: file-icon.svg is static SVG asset used for download file type display
Rule 7: All generated assets are in src/public/images/, not in build/ directory
